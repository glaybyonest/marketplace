# Marketplace

> Full-stack маркетплейс с каталогом, заказами, seller-кабинетом и наблюдаемостью в одном репозитории.

Marketplace - это проект интернет-площадки, в котором уже собраны не только каталог и корзина, но и реальная операционная часть продукта: авторизация, верификация email, восстановление пароля, адреса доставки, оформление заказов, кабинет продавца, админские сценарии, рекомендации, OpenAPI и мониторинг.

Проект решает типичную для marketplace-команд проблему: вместо разрозненных демо и “голого CRUD” здесь есть цельная база для MVP, внутренней платформы, pet-проекта или учебного кейса, где важны и пользовательские сценарии, и эксплуатация, и безопасность.

## Основные возможности

- Каталог товаров с категориями, поисковыми подсказками и популярными запросами.
- Карточка товара, отзывы, избранное, корзина и checkout.
- Личный кабинет покупателя: профиль, адреса, история заказов, активные сессии.
- JWT-auth с refresh-сессиями, email verification, password reset, входом по email/phone-коду.
- Режим cookie-auth с CSRF-защитой как альтернатива bearer-токенам.
- Seller-функции: профиль продавца, dashboard, управление товарами и заказами.
- Admin-функции для управления каталогом и товарами.
- Фоновые задачи для очистки данных, обработки email-очереди и обновления рекомендаций.
- Prometheus, Grafana, health/readiness endpoints и OpenAPI/Swagger.

## Ключевые сценарии использования

- Запустить основу маркетплейса без сборки инфраструктуры с нуля.
- Использовать проект как стартовую платформу для MVP с продавцами и заказами.
- Отрабатывать backend/frontend/DevOps-сценарии в одном репозитории.
- Покрывать QA и регрессию на реальных API-флоу, а не на изолированных моках.

## Для покупателей

- Регистрация, подтверждение email и восстановление пароля.
- Просмотр каталога, поиск, избранное и рекомендации.
- Работа с корзиной и оформление заказа.
- Управление адресами доставки и профилем.
- Контроль активных сессий и отзыв конкретных устройств.

## Для продавцов

- Отдельный seller-раздел в frontend.
- Профиль продавца и данные магазина.
- Создание, обновление, архивирование товаров и управление остатками.
- Просмотр seller dashboard и заказов по своим товарам.

## Почему проект выделяется

- Это не только API: в репозитории уже есть backend, frontend, Docker Compose, OpenAPI и monitoring.
- Авторизация продумана глубже стандартного login/logout: есть refresh-сессии, revoke отдельных сессий, rate limiting, cookie-mode и CSRF.
- Seller- и admin-сценарии встроены в общий продуктовый поток, а не вынесены в отдельный экспериментальный код.
- Фоновые задачи и рекомендации уже встроены в архитектуру, поэтому проект ближе к рабочему сервису, чем к демонстрационному CRUD.
- В репозитории есть не только код, но и практические скрипты для локальной разработки, QA, деплоя и резервного копирования.

## Архитектура по-человечески

Frontend на React/Vite общается с Go API по REST. Backend на Chi и pgx работает с PostgreSQL, поднимает HTTP API, запускает фоновые задачи и публикует метрики. В локальной среде всё собирается через Docker Compose: база, API, Adminer, Prometheus и Grafana стартуют вместе, а OpenAPI доступен прямо из сервиса.

## Быстрый старт

1. Скопируйте примеры env-файлов: `Copy-Item .env.example .env` и `Copy-Item frontend/.env.example frontend/.env`.
2. Запустите локальный стек: `.\start-dev.cmd` или `.\scripts\dev-all.ps1 -OpenBrowser`.
3. Откройте `http://localhost:5173`, API-доки на `http://localhost:8080/docs/` и техдок в [README.tech.md](./README.tech.md).

> Если нужен запуск по частям, миграции, production-compose, backup/restore и разбор модулей, всё вынесено в [README.tech.md](./README.tech.md).

## Текущий статус проекта

- В репозитории уже есть рабочие buyer-, seller- и admin-флоу, Docker-окружение, OpenAPI и observability.
- Email и SMS сейчас отправляются через очередь/логирование; интеграция с внешним SMTP/SMS-провайдером в коде не реализована.

## Что внутри репозитория

| Путь | Что находится внутри |
| --- | --- |
| `cmd/` | Точки входа `api` и `migrate`. |
| `internal/` | Бизнес-логика, HTTP-слой, security, jobs, observability и PostgreSQL-репозитории. |
| `frontend/` | React/Vite/TypeScript клиент с buyer-, seller- и admin-страницами. |
| `migrations/` | Goose-миграции схемы, сидов и служебных таблиц. |

| `docker/` | Инициализация БД, Prometheus, Grafana и Alertmanager-конфиги. |
| `scripts/` | Dev, QA, deploy, backup и restore-сценарии. |
